📚 本地大模型运行指南

基于 Ollama 在 Windows 上运行本地大语言模型

快速开始

1. 安装 Ollama

前往 Ollama 官方网站 下载 Windows 版本并安装。

2. 安装模型

# 安装 qwen2.5-coder:7b 模型（推荐代码生成） ollama pull qwen2.5-coder:7b # 或安装其他模型 ollama pull llama2:7b ollama pull codellama:7b ollama pull mistral:7b

3. 启动服务

# 方法1：前台运行（查看日志） ollama serve # 方法2：后台运行 ollama serve > ollama.log 2>&1 &

4. 基本使用

# 与模型交互 ollama run qwen2.5-coder:7b # 查看已安装模型 ollama list # 查看运行中的模型 ollama ps

Gemma 4 模型实战

1. 关于 Gemma 4

Google Gemma 4 是新一代开源模型，在代码理解和生成方面表现出色，适合编程任务和对话场景。

2. 下载 Gemma 4

# 查看可用的 Gemma 模型 ollama search gemma # 下载 gemma4:e4b 版本（推荐，4B参数，适合8GB显存） ollama pull gemma4:e4b # 或者下载其他版本 ollama pull gemma4 ollama pull gemma4:2b

3. 实战应用

# 启动 gemma4:e4b 模型进行对话 ollama run gemma4:e4b # 示例对话 # >>> 帮我写一个 Python 函数来计算斐波那契数列 # >>> 解释一下什么是递归 # >>> 帮我优化这段代码

4. 切换不同模型

# 方法1：运行时指定模型 ollama run gemma4:e4b # 方法2：查看所有已安装的模型 ollama list # 方法3：使用特定模型回答问题 ollama run qwen2.5-coder:7b # 代码任务用 qwen ollama run gemma4:e4b # 对话任务用 gemma ollama run llama2:7b # 通用任务用 llama

5. 在代码中切换模型

如果你使用 Open WebUI 或 Next Web 等图形界面，可以：在设置中调整默认模型。

# 在 Continue.dev 中切换模型 { "models": [ { "title": "本地 gemma4", "provider": "ollama", "model": "gemma4:e4b", "apiBase": "http://localhost:11434" } ] }

6. Gemma 4 vs 其他模型对比

模型 大小 适用场景 推荐显存
gemma4:e4b ~2.5GB 编程、对话 4GB+
qwen2.5-coder:7b ~4.7GB 代码生成 8GB+
llama2:7b ~3.8GB 通用对话 6GB+

与 VSCode Continue.dev 集成

在 VSCode 的 Continue.dev 扩展设置中添加：

{ "models": [ { "title": "本地 qwen2.5-coder", "provider": "ollama", "model": "qwen2.5-coder:7b", "apiBase": "http://localhost:11434" } ] }

推荐的模型配置

量化版本选择（内存优化）

# Q4_K_M - 平衡版（默认，4.7GB） ollama pull qwen2.5-coder:7b:q4_K_M # Q3_K_M - 更小内存（~3.8GB） ollama pull qwen2.5-coder:7b:q3_K_M # Q5_K_M - 更高精度（~5.2GB） ollama pull qwen2.5-coder:7b:q5_K_M

针对 RTX 3060 Ti 8GB 的优化

  • 模型大小：建议 7B 参数模型
  • 量化等级：Q4_K_M 或 Q3_K_M
  • GPU 层数：50-60 层（在 ~/.ollama/config.json 中配置）

常见问题

1. "Ollama 服务未运行"

启动服务：ollama serve

2. 内存不足

使用更小的量化版本：ollama pull qwen2.5-coder:7b:q3_K_S

3. 生成速度慢

增加 GPU 层数和 CPU 线程数
