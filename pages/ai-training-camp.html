Day 1：AI是什么？能做什么？

🎯 学习目标

  • 理解 AI 辅助编程的原理
  • 知道 AI 能帮你做什么、不能做什么

📖 内容

AI 辅助编程是什么？

AI 编程工具 = 一个 24 小时在线的「高级程序员助手」，它：

  • 能理解你说的话（自然语言）
  • 能读懂你的代码
  • 能帮你写代码、改代码、解释代码
AI 能做什么？

✅ 擅长：

  • 生成重复性代码（CRUD、API 调用）
  • 解释你不熟悉的代码
  • 帮你想思路、给建议
  • 翻译语言（Python → Go）
  • 生成测试用例
  • 优化代码性能

❌ 不擅长：

  • 完全替代你思考
  • 100% 正确的代码（AI 也会犯错）
  • 理解复杂业务逻辑
  • 一次性完成整个大项目
重要原则：你是老板，AI 是助手
你 → 下达指令 → AI → 执行任务 → 你 → 审查结果

你的角色：

  1. 描述清楚需求
  2. 审查 AI 输出的代码
  3. 做最终决策

📝 小作业

  1. 用自己的话解释：什么是 AI 辅助编程？
  2. 列出 3 个你觉得 AI 能帮到你的场景

Day 3：Prompt 基础 — 让 AI 听懂你的话

🎯 学习目标

  • 掌握 Prompt 编写三要素
  • 能写出清晰的指令

📖 内容

为什么 Prompt 很重要？

同样一个工具，写好 Prompt 的人 vs 随便写的人：

  • 输出质量差距：50%+
  • 花费时间差距：2-3 倍
Prompt 三要素
┌────────────────────────────────────┐
│  1️⃣ 角色：你是什么身份？            │
│  2️⃣ 任务：具体要做什么？           │
│  3️⃣ 约束：有什么要求/限制？        │
└────────────────────────────────────┘
模板

❌ 差 Prompt（模糊）

帮我写个排序算法

✅ 好 Prompt（三要素齐全）

请用 Python 实现一个快速排序算法：
1. 角色：你是一位资深算法工程师
2. 任务：实现快速排序算法
3. 约束：
   - 输入：整数列表
   - 输出：排序后的整数列表
   - 时间复杂度 O(n log n)
   - 添加注释说明关键步骤
更多技巧

技巧 1：给示例

请模仿以下风格写一个函数：
[粘贴示例代码]

要求：
- 变量命名清晰
- 有异常处理
- 返回明确的错误信息

技巧 2：分步骤

第一步：请告诉我实现这个功能需要哪些步骤？
第二步：请写出第一步的代码
第三步：请写出第二步的代码

技巧 3：说禁止

请写一个用户登录接口，要求：
- 使用 JWT
- 【禁止】在代码中硬编码密钥
- 【禁止】返回明文密码
- 密码必须加密存储
常见问题

Q：英文 Prompt 效果更好吗？
A：不一定。中文已经很好用，但英文可能在某些场景更精确。

Q：Prompt 多长合适？
A：清晰 > 简短。宁可多写一点描述，也不要让 AI 猜你的意思。

Day 4：实战 — 让 AI 帮你写代码

🎯 学习目标

  • 用 AI 开发一个完整的小功能

📖 内容

实战项目：用 AI 写一个「待办事项」网页

目标：完成后端 API + 前端页面

Step 1：先让 AI 做规划

我要做一个待办事项网页，包含：
- 添加待办
- 标记完成
- 删除待办
技术栈：HTML + CSS + JavaScript（纯前端）

请告诉我：
1. 文件结构
2. 每个文件需要实现的功能
3. 数据存储方案

Step 2：让 AI 写后端

请用 Node.js + Express 实现待办事项的后端 API：
1. GET /todos - 获取所有待办
2. POST /todos - 添加待办
3. PUT /todos/:id - 更新待办
4. DELETE /todos/:id - 删除待办

要求：
- 使用 JSON 文件存储数据
- 返回统一格式 {success: true, data: ...}

Step 3：让 AI 写前端

请帮我写一个待办事项的前端页面：
- 添加输入框 + 添加按钮
- 待办列表，每项有勾选框和删除按钮
- 点击待办可以编辑

要求：
- 纯 HTML + CSS + JS
- 美观简洁
- 适配手机端

Day 6：进阶 — 本地部署大模型

🎯 学习目标

  • 在本地电脑运行自己的 AI 模型
  • 理解什么场景适合本地部署

📖 内容

为什么要本地部署？

优点：

  • ✅ 数据不外泄（隐私代码安全）
  • ✅ 无限使用（不花钱）
  • ✅ 离线可用

缺点：

  • ⚠️ 需要好显卡（至少 4GB 显存）
  • ⚠️ 速度比在线 API 慢
适合人群
场景推荐
代码隐私敏感（公司项目）本地部署
学习/练习/个人项目云端 API
低配电脑先用云端
Ollama 安装教程

Step 1：下载 Ollama

官网：https://ollama.com
下载 Windows 版本

Step 2：安装
双击安装包，默认安装

Step 3：下载模型

打开命令行（cmd 或 PowerShell）
输入以下命令下载模型：

ollama pull llama3
（首次下载约 4-5GB，等待几分钟）

或者下载更小的模型（适合低配电脑）：
ollama pull codellama:7b

Step 4：测试运行

ollama run llama3

输入你想问的问题，测试是否正常
在 Cursor 中使用本地模型
  1. 安装 Continue 插件（VSCode）
  2. 配置本地模型：
{
  "languageModel": {
    "ollama": {
      "host": "http://localhost:11434",
      "model": "llama3"
    }
  }
}

  1. 安装 Ollama（可选，有好显卡再装）
  2. 运行一个本地模型测试
  3. 如果成功，在 VSCode 中配置使用
