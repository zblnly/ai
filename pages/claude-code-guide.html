🖥️ Claude Code 完全使用指南

Anthropic 官方推出的终端原生 AI 编程助手 · 2026 最新版

什么是 Claude Code？

Claude Code 是 Anthropic 推出的终端原生 AI 编程助手，运行在命令行中。它能读取整个项目、修改代码、运行 Git 命令、执行测试，像一个"会写代码的 AI 同事"一样与你协作。

  • 200K 超长上下文 — 轻松驾驭大型代码库
  • 支持 40+ 编程语言 — TypeScript、Python、Go、Rust、Java 等主流语言和框架
  • 终端原生 — 直接在你的开发环境中运行，不需要切换到网页或 IDE
  • 安全沙箱 — 默认隔离运行，不触碰你的文件直到你授权
  • 可替换模型 — 支持接入第三方 API，国内用户可配置中转

📦 安装 Claude Code

方式一：原生安装（推荐，无需 Node.js）

macOS / Linux：

curl -fsSL https://claude.ai/install.sh | bash

Windows PowerShell：

irm https://claude.ai/install.ps1 | iex

Homebrew（Mac）：

brew install --cask claude-code

方式二：npm 全局安装

npm install -g @anthropic-ai/claude-code

前置要求：Node.js 18+、npm 9+、Git 2.30+。

验证安装

claude --version

安装成功后，在项目目录下运行 claude 即可启动。

🚀 快速上手

  1. 进入项目目录：cd your-project
  2. 启动 Claude Code：claude
  3. 首次使用需完成 OAuth 认证（浏览器自动打开）
  4. 输入你的需求，例如："帮我理解这个项目的结构"

启动后你会进入一个交互式对话界面，直接输入自然语言即可。Claude Code 会自动读取你的项目文件来理解上下文。

⚡ 核心功能

CLAUDE.md — 项目上下文文件

在项目根目录创建 CLAUDE.md 文件，记录编码规范、项目架构、常用命令等信息。Claude Code 每次启动自动读取，作为系统指令执行。

# CLAUDE.md

# 项目概述
AI学习资源站，纯静态网站，GitHub Pages 部署

# 编码规范
- 所有页面使用中文
- CSS 使用 CSS 变量定义的色板
- 不引入外部依赖框架

# 常用命令
- npm run build    # 构建项目
- npm run dev      # 启动开发服务器

可以用 /init 命令让 Claude Code 自动分析项目并生成 CLAUDE.md。

斜杠命令

命令功能说明
/help查看所有可用命令
/init分析项目并生成 CLAUDE.md
/model <名称>切换模型（Opus / Sonnet / Haiku）
/clear清除对话历史，释放上下文
/compact压缩上下文，保留关键信息后继续对话
/config打开配置面板
/permissions管理工具权限
/cost查看当前会话的 Token 消耗
/ide连接 VS Code / JetBrains IDE
/review审查代码变更（PR Review）
/security-review安全审查当前分支改动

权限系统

Claude Code 的所有文件写入、命令执行、网络请求都需要你的明确授权。你可以在 .claude/settings.json 中配置权限规则：

{
  "permissions": {
    "allow": [
      "Bash(npm test)",
      "Bash(npm run lint)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force origin main)"
    ]
  }
}

Hooks（钩子）

在关键事件节点自动执行脚本，例如提交前运行 lint、代码修改后自动格式化等。在 settings.json 中配置：

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "command": "npx eslint --fix $FILE"
      }
    ]
  }
}

MCP 扩展（模型上下文协议）

通过 MCP 服务器连接外部工具，让 Claude Code 访问数据库、浏览器、Jira 等。MCP 服务商目前有 mcp.so、smithery.ai 等第三方网站可以浏览发现。

Sub-agent（子代理）

Claude Code 可以启动子代理处理独立任务，如代码库调研、代码审查等，节省约 40% 的输入 Token。

🔌 IDE 集成

VS Code

  1. 安装 VS Code 扩展：搜索 "Claude Code"
  2. 在终端中运行 claude，输入 /ide
  3. VS Code 将自动连接到当前会话

连接后，Claude Code 可以直接在编辑器中打开文件、显示 diff、应用修改。

JetBrains（IntelliJ IDEA / PyCharm 等）

安装 JetBrains 插件后，同样通过 /ide 命令连接。

🇨🇳 国内用户配置

由于 Claude 官方 API 在国内无法直接访问，需要配置中转 API。编辑 ~/.claude/settings.json：

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://你的中转站地址",
    "ANTHROPIC_AUTH_TOKEN": "你的API-Key",
    "ANTHROPIC_MODEL": "claude-sonnet-4-6"
  }
}

同时创建 ~/.claude/.claude.json 跳过官方验证：

{ "hasCompletedOnboarding": true }

可接入的模型包括 GLM、Qwen、MiniMax 等国产大模型，也可以使用阿里云百炼、百度千帆等平台提供的 Claude 服务。

💡 最佳实践

1. 写好 CLAUDE.md

这是最重要的第一步。控制在 200 行以内，只写 Claude 无法从代码中推断出 的信息。包括：项目是干什么的、用什么技术栈、代码风格偏好、禁止做的事、常用命令。

2. 用规划模式处理复杂任务

对于多文件修改或涉及架构决策的任务，先按 Shift+Tab 进入规划模式，让 Claude 先出方案再执行。简单修 bug 则跳过规划。

3. 保持上下文清洁

  • 当对话变长或思路混乱时，用 /clear 重置
  • 收到 2 次纠正后建议用 /clear 清除，避免 AI 在"错误的上下文"里打转
  • 一个任务一个会话，不要在一个会话里混着做多件事

4. 创建 .claudeignore 文件

排除不需要分析的文件，节省上下文空间：

node_modules/
.git/
dist/
build/
*.min.js
*.lock
package-lock.json
yarn.lock

5. 模型选择策略

  • Sonnet — 日常开发，速度快、成本低
  • Opus — 复杂架构设计、深度推理任务（虽然单价高，但往往一次就做对，总成本反而更低）
  • Haiku — 简单查询、格式化等低成本任务

6. 使用 Sub-agent 做调研

让子代理去搜索代码库、查找定义，比自己手动搜索省 40% 以上的 Token。

🔧 常见问题与排查

问题解决方案
Windows 提示 "bash 不是内部命令" 安装 Git for Windows，确保 Git Bash 可用
返回 401 认证错误 检查 API Key 格式，确认 Base URL 正确，重新运行 claude 进行 OAuth 认证
"thinking type" 400 错误 在环境变量中设置 CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1
环境变量不生效 设置环境变量后必须重启终端
上下文过长 / 响应变慢 使用 /compact 压缩上下文或 /clear 重新开始
npm 全局安装权限报错（Linux/Mac） 使用 sudo npm install -g 或配置 npm prefix 到用户目录
