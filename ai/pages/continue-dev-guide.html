📖 什么是Continue.dev？

Continue.dev 是一款开源的AI编程助手扩展，支持VS Code和JetBrains IDE，可连接任意LLM提供商。

与GitHub Copilot等闭源工具不同，它最大的优势在于：

  • 完全开源 - 25K+ GitHub Stars
  • 模型无关 - 支持任意LLM（OpenAI、Claude、Ollama等）
  • 本地部署 - 可使用本地模型，保护代码隐私
  • 深度集成 - 四种核心模式：Autocomplete、Edit、Chat、Agent

🚀 快速开始

第一步：安装扩展

  1. 打开VS Code，进入扩展市场（Ctrl+Shift+X）
  2. 搜索 "Continue"
  3. 点击安装

VSCode扩展市场 →

第二步：配置模型

安装后，Continue会提示你选择模型提供商：

# 方式1：使用Ollama本地模型（推荐） ollama pull qwen2.5-coder:7b # 方式2：使用OpenAI API 需要在config.json中配置API Key # 方式3：使用Claude API 需要在config.json中配置API Key

第三步：开始使用

按Ctrl+L打开侧边栏，开始与AI对话。

🎯 四大核心功能

1. Autocomplete（自动补全）

在编写代码时提供智能的inline建议。

// TODO: Implement a sorting algorithm function sortingAlgorithm(arr) { // AI会自动建议实现 }

按Tab接受建议，Esc拒绝。

2. Edit（编辑模式）

对选中的代码进行快速修改。

  1. 选中需要修改的代码
  2. 按Ctrl+Shift+I或右键选择"Edit"
  3. 输入修改指令
  4. AI会显示diff预览，接受或拒绝

3. Chat（对话模式）

与AI进行交互式对话。

  • 按Ctrl+L打开侧边栏
  • 选择代码后提问
  • 适合代码审查、调试、学习新概念

4. Agent（智能代理）

自主完成复杂任务的AI代理。

  • 切换到Agent模式
  • 输入复杂任务如："创建单元测试"
  • AI会分析代码、创建文件、处理导入
# 示例任务 "Write comprehensive unit tests for the sorting functions in this file"

💡 常用场景

场景一：快速理解陌生代码库

  1. 选中一段复杂代码
  2. 按Ctrl+L打开侧边栏
  3. 输入："解释这段代码的作用，并指出潜在问题"
# 示例对话 你：解释这个函数的作用，并指出潜在问题 AI：[详细解释代码逻辑，并给出改进建议]

场景二：代码重构

  1. 选中需要重构的代码块
  2. 输入："重构这段代码，使其更符合最佳实践"

场景三：生成单元测试

  1. 选中目标函数
  2. 输入："为这个函数生成完整的单元测试，覆盖边界情况"

场景四：Debug排错

  1. 粘贴报错信息
  2. 描述你已尝试过的方法
  3. 让AI分析可能的原因
# 示例 [Error: Cannot read property 'name' of undefined] 我已经尝试过： 1. 检查了数据库连接，正常 2. 重启了服务，问题依旧 请分析可能的原因，按可能性从高到低排序。

🔧 高级技巧

技巧一：使用 @ 符号引用文件

引用项目中的文件作为上下文：

@filename.py - 引用特定文件 @folder/ - 引用整个文件夹 @codebase - 引用整个代码库
@utils.py @models.py 请帮我重构这两个文件的导入语句，消除循环依赖

技巧二：配置自定义指令

在config.json中添加systemPrompt：

{ "systemPrompt": "你是一位资深的Python开发者。请遵循以下规范：\n- 使用type hints\n- 遵循PEP 8风格指南\n- 优先使用函数式编程\n- 所有公共函数必须有docstring" }

技巧三：使用模板快速生成

用斜杠命令快速生成代码：

  • /edit - 编辑选中代码
  • /comment - 添加注释
  • /test - 生成测试
  • /fix - 修复问题

在侧边栏输入/查看所有命令。

技巧四：多模型切换

不同任务使用不同模型：

任务 推荐模型
代码生成 Claude 3.5 / GPT-4o
代码审查 Claude（擅长分析）
快速补全 本地Ollama（低延迟）
复杂推理 Claude Opus

技巧五：自定义命令

添加常用工作流命令：

{ "customCommands": [ { "name": "review", "prompt": "审查此代码以查找错误、安全问题和性能问题。要具体且可执行。", "description": "代码审查" }, { "name": "test", "prompt": "为此代码编写全面的单元测试。包含边界情况。", "description": "生成测试" }, { "name": "docstring", "prompt": "按照最佳实践添加文档字符串。", "description": "添加文档" }, { "name": "optimize", "prompt": "优化此代码以提高性能。", "description": "优化代码" } ] }

技巧六：隐私保护

处理敏感代码时：

  1. 使用本地模型（Ollama、LM Studio）
  2. 配置私有部署的LLM服务
  3. 避免在对话中粘贴敏感信息

❓ 常见问题

Q1: Continue和GitHub Copilot有什么区别？

主要区别：开源vs闭源，模型自由选择vs绑定特定模型，支持本地模型vs必须联网。

Q2: 需要付费吗？

基础功能免费，高级功能需要付费。也可以使用免费模型（Ollama本地模型）。

Q3: 连接被拒绝怎么办？

检查Ollama是否启动：ollama serve或curl http://localhost:11434

Q4: 如何更新扩展？

VS Code: 扩展市场 → 搜索Continue → 点击更新
JetBrains: Settings → Plugins → 更新
